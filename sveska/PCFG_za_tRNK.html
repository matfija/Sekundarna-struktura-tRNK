PCFG za tRNK¶

U radu je razmotrena primena probabilistikih kontekstno slobodnih gramatika (PCFG) u predvianju sekundarne strukture transportne ribonukleinske kiseline (tRNK). Preuzet je skup tRNK sa pridruenim strukturama, nakon ega je na delu podataka obueno nekoliko stohastikih gramatika. Modeli su vrednovani prema ponaanju na drugom delu podataka, odnosno slinosti predviene strukture sa stvarnom. Uporeeni su kako meusobno, tako i sa literaturom.

1. Primarna struktura ⮭¶

Sekundarnoj strukturi, naravno, prethodni primarna. RNK se najee predstavlja kao niska nad azbukom ribonukleotida $\{A, U, C, G\}$.

Pored $A-U$ i $C-G$ prema Votsonu i Kriku, zbog povoljne termodinamike konfiguracije ponekad se uparuje i nekanonski (wobble) par $U-G$.

In [1]:
# Azbuka ribonukleotida
A, U, C, G = 'AUCG'

# Pamenje azbuke kao skupa
azbuka = {A, U, C, G}

# Mogua uparivanja ribonukleotida
parovi = [(A, U), (U, A), (C, G), (G, C), (U, G), (G, U)]

# Provera korektnosti uparivanja
def par(*rns):
    return rns in parovi

Specijalno, sekvencu mogu initi i drugi karakteri, koji predstavljaju modifikovane baze. Na primer, pseudouridin (P) predstavlja izmenjeni uridin (U). Jednostavnosti radi, a usled ogranienja uparivanja koje sama sekundarna struktura namee, ovakva modifikovanost moe se slobodno zanemariti.

In [4]:
# Biblioteka za regularne izraze
import re

# Svoenje sekvence na etiri ribonukleotida
def standardizuj(sekvenca):
    # Standardizacija svih modifikacija adenina
    sekvenca = re.sub('[H"/+*=6E[:IO^`]', A, sekvenca)
    
    # Standardizacija svih modifikacija uracila
    sekvenca = re.sub(r'[tN{2J4&1S3V5!$X,)~DP\]ZTF\\]', U, sekvenca)
    
    # Standardizacija svih modifikacija citozina
    sekvenca = re.sub("[<%BM?'}>]", C, sekvenca)
    
    # Standardizacija svih modifikacija guanina
    sekvenca = re.sub('[;KL#R|7(Q89YW]', G, sekvenca)
    
    # Vraanje standardizovane sekvence
    return sekvenca

2. Sekundarna struktura ⮭¶

Transportna RNK ima karakteristinu sekundarnu strukturu u obliku deteline sa tri lista. Listove (petlje, loop) ine neuparene baze, dok se kao veza izmeu njih nalaze etiri zavojnice (drke, stem) i umetnuti nukleotidi. Na drugom listu, otprilike u sredini sekvence, nalazi se antikodon aminokiseline koja se prenosi.

I sekundarnu strukturu mogue je predstaviti kao nisku, i to nad azbukom koja se sastoji od take i zagrada. U ovakvoj notaciji, taka predstavlja neuparenu bazu, dok zagrade oznaavaju parove. Uparivanje baza, oekivano, prati matematiko i jeziko pravilo uparivanja zagrada, uz dodatni mininalni razmak.

In [9]:
# Azbuka strukture
zagrade = {'(', '.', ')'}

# Provera da li je struktura korektna
def proveri_strukturu(struktura, razmak=0):
    # Inicijalizacija uparivanja
    uparivanje = {}
    
    # Provera da li je azbuka dobra
    if not set(struktura) <= zagrade:
        return False, uparivanje
    
    # Inicijalizacija steka indeksa
    stek = []
    
    # Prolazak kroz celu strukturu
    for j, c in enumerate(struktura):
        # Stavljanje prvog lana uparivanja na stek
        if c == '(':          
            stek.append(j)
        
        # Provera da li je stek neprazan
        elif c == ')':
            if not stek:
                return False, uparivanje
            
            # Dohvatanje prvog lana uparivanja
            i = stek.pop()
            
            # Provera korektnosti razmaka uparivanja
            if j - i <= razmak:
                return False, uparivanje
            
            # Uparivanje prvog i drugog lana
            uparivanje[i] = j
    
    # Provera da li je stek prazan
    return not stek, uparivanje

# Provera saglasnosti sekvence i strukture
def proveri_uparenost(sekvenca, struktura, razmak=0, uparivanje=None):
    # Inicijalizacija mape broja parova
    uparenost = {rn_i: {rn_j: 0 for rn_j in azbuka} for rn_i in azbuka}
    
    # Sekvenca i struktura moraju biti iste duine
    if len(sekvenca) != len(struktura):
        return False, uparenost
    
    # Odreivanje uparivanja ukoliko nije poznato
    if uparivanje is None:
        ok, uparivanje = proveri_strukturu(struktura, razmak)
        
        # Provera da li je struktura dobra
        if not ok: return False, uparenost
    
    # Prolazak kroz sva uparivanja
    for i, j in uparivanje.items():
        rn_i = sekvenca[i]
        rn_j = sekvenca[j]
        
        # Nukleotidi moraju biti korektan par
        if not par(rn_i, rn_j):
            return False, uparenost
        
        # Uveavanje brojaa parova
        uparenost[rn_i][rn_j] += 1
    
    # Vraanje mape broja parova
    return True, uparenost

# Biblioteka za rad sa stablima
from nltk import Tree

# Biblioteka za crtanje
from ipynb.draw import draw_struct

# Primarna sekvenca sa pridruenom sekundarnom strukturom
class Struktura2D:
    def __init__(self, sekvenca, struktura, razmak=2):
        # Provera i standardizacija sekvence
        self.proveri_sekvencu(sekvenca)
        
        # Provera strukture i uparivanja
        self.proveri_strukturu(struktura, razmak)
        
        # Provera tanog uparivanja baza
        self.proveri_uparenost()
        
        # Odreivanje drveta izvoenja sekvence
        self.drvo = self.S(0, self.n - 1)
    
    # SVG prikaz drveta izvoenja sekvence
    def _repr_svg_(self):
        return self.drvo._repr_svg_()
    
    # Provera i standardizacija sekvence
    def proveri_sekvencu(self, sekvenca):
        ok, self.sekvenca = proveri_sekvencu(sekvenca)
        
        # Izuzetak ako neto nije u redu
        if not ok: raise ValueError
        
        # uvanje duine sekvence
        self.n = len(sekvenca)
    
    # Provera strukture i uparivanja
    def proveri_strukturu(self, struktura, razmak):
        ok, self.uparivanje = proveri_strukturu(struktura, razmak)
        
        # Izuzetak ako neto nije u redu
        if not ok: raise ValueError
        
        # uvanje razmaka
        self.razmak = razmak
        
        # uvanje strukture
        self.struktura = struktura
    
    # Provera tanog uparivanja baza
    def proveri_uparenost(self):
        ok, self.uparenost = proveri_uparenost(self.sekvenca, self.struktura,
                                               self.razmak, self.uparivanje)
        
        # Izuzetak ako neto nije u redu
        if not ok: raise ValueError
    
    # Drvo bazne klase je prost niz strukturiranih baza
    def S(self, *args):
        return Tree('S', zip(self.sekvenca, self.struktura))
    
    # Crtanje enkapsulirane sekundarne strukture
    def nacrtaj(self):
        draw_struct(self.sekvenca, self.struktura)

3. Gramatika Nussinov ⮭¶

Najjednostavnije kontekstno slobodne gramatike za modelovanje strukture RNK nadahnute su algoritmom Nusinov za maksimizaciju broja uparivanja, objavljenim 1980. Ukoliko su dozvoljena prazna (epsilon) pravila izvoenja, mogua je sledea kompaktna forma: $S dSd | SS | s | \epsilon$. Pritom $d$ oznaava uparene baze, $s$ neuparene, dok $\epsilon$ predstavlja kraj izvoenja. Pravilo $S SS$ omoguava grananje, a time i postojanje veeg broja zavojnica i petlji.

Usled ogranienja koriene biblioteke za rad sa stohastikim kontekstno slobodnim gramatikama, implementirana je verzija bez praznih pravila. Epsilon izvoenja, naime, onemoguavaju primenu ugraene klase za maksimizaciju verovatnoe drveta. Konana gramatika je: $S dSd$ (uparivanje) $| SS$ (grananje) $| s$ (neuparena baza). Minimalna duina petlje je jedan, a vieznanost grananja (odluka gde tano dolazi do preseka) programatski je razreena.

In [23]:
# Kontekstno slobodna gramatika Nussinov
class Nussinov(Struktura2D):
    def __init__(self, sekvenca, struktura):
        super().__init__(sekvenca, struktura, 1)
    
    # Obrada neterminala S[i:j]
    def S(self, i, j):
        # Najlevlji (tekui) ribonukleotid
        rn_i = self.sekvenca[i]
        
        # Jedan karakter mora biti neuparena baza
        # na zavrnoj poziciji: S[i:j]  s
        if i == j:
            return Tree('S', [rn_i])
        
        # Neuparena baza je u suprotnom u petlji,
        # pa je izvoenje S[i:j]  S[i:i] S[i+1:j]
        if self.struktura[i] == '.':
            return Tree('S', [self.S(i, i), self.S(i+1, j)])
        
        # Odreivanje uparene baze
        k = self.uparivanje[i]
        rn_k = self.sekvenca[k]
        
        # Potpuno uparivanje znai da je izvoenje
        # u zavojnici: S[i:j]  d S[i+1:j-1] d
        if j == k:
            return Tree('S', [rn_i, self.S(i+1, j-1), rn_k])
        
        # Nepotpuno uparivanje uslovljava grananje,
        # pa je izvoenje S[i:j]  S[i:k] S[k+1:j]
        return Tree('S', [self.S(i, k), self.S(k+1, j)])

Kako su strukture predstavljene kao niske taaka i zagrada, neophodno je transformisati ih u odgovarajua stabla izvoenja prema gramatici. Upravo to je i uinjeno modifikovanom tehnikom rekurzivnog spusta. Enkapsulacija anotirane RNK gramatikom, dakle, podrazumeva izraunavanje pretpostavljenog drveta.

4. Gramatika KH-99 ⮭¶

Iako su kontekstno slobodne gramatike, sasvim prirodno, osloboene od konteksta, on im se moe donekle pridodati paljivo smiljenim pravilima izvoenja. Primera radi, pravilo $S kS$ moe se prepisati kao $S kS^k$. Novi neterminal $S^k$ uvodi se za svaki terminal $k$ i u sutini poznaje svoj kontekst: u pitanju je $S$ koje se nalazi upravo iza simbola $k$. Ovakvo uslonjavanje gramatike naziva se markovizacija i predstavlja jedan od naina za poveavanje opisne moi.

Markovizacija, meutim, moe proizvesti vrlo sloenu gramatiku, a da pritom ne donese srazmerno znaajnu dobit. Stoga se esto pribegava neto jednostavnijim metodama za dodavanje konteksta. Jedna takva gramatika za predvianje sekundarne strukture RNK zasnovana je na algoritmu KH-99, koji su 1999. predstavili Knudsen i Hejn. Pored poznatih simbola gramatike Nusinov, ova gramatika poznaje i neterminale $L$ i $F$, koji joj upravo pridodaju kontekst.

$$ \large S LS \scriptsize\text{ (nizanje elemenata) } \large| L \scriptsize\text{ (poslednji element) } $$$$ \large L s \scriptsize\text{ (neuparena baza) } \large| dFd \scriptsize\text{ (poetak zavojnice) } $$$$ \large F dFd \scriptsize\text{ (nastavak zavojnice) } \large| LS \scriptsize\text{ (unutranjost zavojnice) } $$

Primetno je da se uparivanja ocenjuju afino: razliito ona na poetku zavojnice i ona unutar nje. Na taj nain uveden je osnovni, ali znaajan kontekst, to se ogleda u dobrim rezultatima. Minimalna duina petlje je dva, sa izvoenjem $F LS sS sL ss$, a gramatika je jednoznana za poznatu strukturu. Autori su, inae, 2003. kroz softver Pfold proirili i poboljali svoj metod ukljuivanjem u priu evolutivnih informacija o razmatranim sekvencama.

# Kontekstno slobodna gramatika KH-99
class KH99(Struktura2D):
    def __init__(self, sekvenca, struktura):
        super().__init__(sekvenca, struktura)
    
    # Obrada startnog neterminala S[i:j]
    def S(self, i, j):
        # Jedan karakter mora biti neuparena baza
        # pa je izvoenje S[i:j]  L[i:j]
        if i == j:
            return Tree('S', [self.L(i, j)])
        
        # Neuparena baza je u suprotnom u petlji,
        # pa je izvoenje S[i:j]  L[i:i] S[i+1:j]
        if self.struktura[i] == '.':
            return Tree('S', [self.L(i, i), self.S(i+1, j)])
        
        # Odreivanje uparene baze
        k = self.uparivanje[i]
        
        # Potpuno uparivanje znai da je izvoenje
        # u nekoj zavojnici: S[i:j]  L[i:j]
        if j == k:
            return Tree('S', [self.L(i, j)])
        
        # Nepotpuno uparivanje uslovljava grananje,
        # pa je izvoenje S[i:j]  L[i:k] S[k+1:j]
        return Tree('S', [self.L(i, k), self.S(k+1, j)])
    
    # Obrada neterminala petlje L[i:j]
    def L(self, i, j):
        # Najlevlji (tekui) ribonukleotid
        rn_i = self.sekvenca[i]
        
        # Jedan karakter mora biti neuparena baza,
        # pa je izvoenje tipa L[i:j]  s
        if i == j:
            return Tree('L', [rn_i])
        
        # Odreivanje uparene baze
        rn_j = self.sekvenca[j]
        
        # Potpuno uparivanje znai da je izvoenje
        # u zavojnici: L[i:j]  d F[i+1:j-1] d
        return Tree('L', [rn_i, self.F(i+1, j-1), rn_j])
    
    # Obrada neterminala zavojnice F[i:j]
    def F(self, i, j):
        # Neuparena baza je u unutranjosti heliksa,
        # pa je izvoenje F[i:j]  L[i:i] S[i+1:j]
        if self.struktura[i] == '.':
            return Tree('F', [self.L(i, i), self.S(i+1, j)])

        # Najlevlji (tekui) ribonukleotid
        rn_i = self.sekvenca[i]
        
        # Odreivanje uparene baze
        k = self.uparivanje[i]
        rn_k = self.sekvenca[k]
        
        # Potpuno uparivanje znai da je izvoenje
        # u zavojnici: F[i:j]  d F[i+1:j-1] d
        if j == k:
            return Tree('F', [rn_i, self.F(i+1, j-1), rn_k])
        
        # Nepotpuno uparivanje uslovljava grananje,
        # pa je izvoenje F[i:j]  L[i:k] S[k+1:j]
        return Tree('F', [self.L(i, k), self.S(k+1, j)])

5. Model kovarijacije ⮭¶

Posebnu primenu u modelovanju familija RNK i njihove strukture imaju modeli kovarijacije (CM). Re je o gramatikama ija pravila izvoenja podrazumevaju postojanje dobro ouvanog skeleta strukture u familiji koja se modeluje. Ovakva pravila, oekivano, nose sa sobom najvie konteksta. Najpoznatija baza kovarijacionih modela sa pridruenim viestrukim poravnanjima, saglasnim strukturama i mnogim metodama analize i vizuelizacije jeste Rfam. Sadri, naravno, i podatke o tRNK. Dodatan aktuelni resurs za rad i pretragu baza podataka pomou kovarijansnih modela tRNK jeste tRNAscan-SE 2.0.

Ve je pomenuto da je sekundarna struktura transportnih RNK karakteristina po tome to ima oblik deteline sa tri lista. Postoje, dakle, tano tri vane petlje (redom $D$, $Ac$, $T$), tano etiri zavojnice (redom $Acc$, $D$, $Ac$, $T$) i jo neki dodatni elementi ($V$ petlja, $CCA$ rep). tavie, mnogi od ovih elemenata su fiksne duine (npr. $Acc$ drka ima est ili sedam parova), pa ak i sadraja (npr. $CCA$ rep, ukoliko postoji), to tRNK ini posebno pogodnom za konstrukciju CM.

$$ \large G s \scriptsize\text{ (glava je tano jedan nukleotid) } $$$$ \large R Gcca \scriptsize\text{ (glava i $CCA$ rep) } \large| G \scriptsize\text{ (samo glava) } $$$$ \large S GZ^SR \scriptsize\text{ (glava, zavojnica i rep) } \large| Z^SR \scriptsize\text{ (zavojnica i rep) } $$$$ \large Z^S dZ^Sd \scriptsize\text{ (uparivanje) } \large| GZ^SG \scriptsize\text{ (promaaj) } \large| GGZ^UZ^ULZ^U \scriptsize\text{ (unutranjost) } $$$$ \large Z^U dZ^Ud \scriptsize\text{ (uparivanje) } \large| GZ^UG \scriptsize\text{ (promaaj) } \large| L \scriptsize\text{ (unutranjost) } $$$$ \large L GL \scriptsize\text{ (nizanje baza) } \large| GG \scriptsize\text{ (dve baze) } $$

Ovakva gramatika nije pravi kovarijacioni model po definiciji, ali ipak prilino precizno oslikava strukturu transportne RNK. Prednost joj je to za obuavanje ne zahteva poznavanje viestrukog poravnanja trening skupa, mada je kao posledica toga neto manje uspena. Jednoznana je za poznatu strukturu, uz napomenu da zahteva poznavanje cele sekvence u svakom trenutku. Minimalna duina petlje je dva, sa izvoenjem $Z^U L GG sG ss$.

# Model kovarijacije tRNK
class CM(Struktura2D):
    def __init__(self, sekvenca, struktura):
        super().__init__(sekvenca, struktura)
    
    # Obrada startnog neterminala S[i:j]
    def S(self, i, j):
        # Broj najdenjih neuparenih baza
        n = j - self.struktura.rfind(')')
        
        # Odreivanje granice zavojnice i repa
        k = j - n + int(n == 2 or n == 5)
        
        # Broj najlevljih neuparenih baza
        m = self.struktura.find('(')
        
        # Ako je viak neuparenih jednak sa obe strane, nema
        # glave, pa je izvoenje S[i:j]  Z^S[i:k] R[k+1:j]
        if m == int(n == 2 or n == 5):
            return Tree('S', [self.ZS(i, k), self.R(k+1, j)])
        # U suprotnom postoji glava, strogo jednolana, pa je
        # izvoenje tipa S[i:j]  G[i:i] Z^S[i+1:k] R[k+1:j]
        else:
            return Tree('S', [self.G(i, i), self.ZS(i+1, k), self.R(k+1, j)])
    
    # Obrada neterminala glave G[i:j]
    def G(self, i, j):
        # Najlevlji (tekui) ribonukleotid
        rn_i = self.sekvenca[i]
        
        # Izvoenje je uvek tipa G[i:j]  s
        return Tree('G', [rn_i])
    
    # Obrada neterminala repa R[i:j]
    def R(self, i, j):
        # Jedan karakter mora biti glava,
        # pa je izvoenje R[i:j]  G[i:j]
        if i == j:
            return Tree('R', [self.G(i, j)])
        
        # U suprotnom je CCA rep, pa je izvoenje
        # tipa R[i:j]  G[i:i] 'C' 'C' 'A'
        return Tree('R', [self.G(i, i), 'C', 'C', 'A'])
    
    # Odreivanje kraja zavojnice
    def zavojnica(self, i, j):
        # Prolazak kroz sve indekse
        for l in range(i, j):
            # Odabir prvog uparivanja
            if self.struktura[l] == '(':
                k = self.uparivanje[l]
                
                # Dodavanje broja neuparenih
                return k + l - i
    
    # Obrada neterminala spoljanje
    # zavojnice (drke) Z^S[i:j]
    def ZS(self, i, j):
        # Zavojnica se potencijalno nastavlja
        if self.struktura[i] == '(':
            # Granini (tekui) ribonukleotidi
            rn_i = self.sekvenca[i]
            rn_j = self.sekvenca[j]
            
            # Izvoenje je Z^S[i:j]  d Z^S[i+1:j-1] d
            return Tree('Z^S', [rn_i, self.ZS(i+1, j-1), rn_j])
        
        # Zavojnica potencijalno ima prekid, pa je izvoenje
        # tipa Z^S[i:j]  G[i:i] Z^S[i+1:j-1] G[j:j]
        if self.struktura[i+1] == '(':
            return Tree('Z^S', [self.G(i, i), self.ZS(i+1, j-1), self.G(j, j)])
        
        # U suprotom se stiglo do unutranjosti, koja ima sloeno izvoenje tipa
        # Z^S[i:j]  G[i:i] G[i+1:i+1] Z^U[i+2:k] Z^U[k+1:l] L[l+1:m-1] Z^U[m:j]
        
        # Odreivanje slobodnih baza
        g1 = self.G(i, i)
        g2 = self.G(i+1, i+1)
        
        # Odreivanje prve zavojnice
        k = self.zavojnica(i+2, j)
        z1 = self.ZU(i+2, k)
        
        # Odreivanje druge zavojnice
        l = self.zavojnica(k+1, j)
        z2 = self.ZU(k+1, l)
        
        # Odreivanje poetka tree zavojnice
        for m in range(l+2, j):
            if self.struktura[m] == '(':
                break
        
        # Pomeranje poetka ako ima neuparenih
        m -= j - self.uparivanje[m]
        
        # Odreivanje tree zavojnice
        z3 = self.ZU(m, j)
        
        # Odreivanje varijabilne petlje
        l = self.L(l+1, m-1)
        
        # Primena izvoenja unutranjosti, ali u prilagoenoj binarnoj formi:
        # Z^S[i:j]  G[i:i] Z^S1[i+1:j], Z^S1[i+1:j]  G[i+1:i+1] Z^S2[i+2:j],
        # Z^S2[i+2:j]  Z^U[i+2:k] Z^S3[k+1:j], Z^S3[k+1:j]  Z^U[k+1:l] Z^S4[l+1:j],
        # Z^S4[l+1:j]  L[l+1:m-1] Z^U[m:j], koja je neophodna zbog algoritma
        return Tree('Z^S', [g1, Tree('Z^S1', [g2, Tree('Z^S2', [z1,
                    Tree('Z^S3', [z2, Tree('Z^S4', [l, z3])])])])])
    
    # Obrada neterminala unutranjih
    # zavojnica (drki) Z^U[i:j]
    def ZU(self, i, j):
        # Zavojnica se potencijalno nastavlja
        if self.struktura[i] == '(':
            # Granini (tekui) ribonukleotidi
            rn_i = self.sekvenca[i]
            rn_j = self.sekvenca[j]
            
            # Izvoenje je Z^U[i:j]  d Z^U[i+1:j-1] d
            return Tree('Z^U', [rn_i, self.ZU(i+1, j-1), rn_j])
        
        # Zavojnica potencijalno ima prekid, pa je izvoenje
        # tipa Z^U[i:j]  G[i:i] Z^U[i+1:j-1] G[j:j]
        if self.struktura[i+1] == '(':
            return Tree('Z^U', [self.G(i, i), self.ZU(i+1, j-1), self.G(j, j)])
        
        # U suprotnom se stiglo do petlje, pa
        # je izvoenje tipa Z^U[i:j]  L[i:j]
        return Tree('Z^U', [self.L(i, j)])
    
    # Obrada neterminala petlje L[i:j]
    def L(self, i, j):
        # Dva karaktera moraju biti bazni sluaj,
        # pa je izvoenje L[i:j]  G[i:i] G[j:j]
        if i + 1 == j:
            return Tree('L', [self.G(i, i), self.G(j, j)])
        
        # U suprotno se baze niu, tako da je
        # izvoenje tipa L[i:j]  G[i:i] L[i+1:j]
        return Tree('L', [self.G(i, i), self.L(i+1, j)])

# itanje podataka o tRNK
def podaci(fst):
    with open(fst, encoding='utf-8') as f:
        fst = f.read().split()
    
    # Broj tRNK u skupu podataka
    n = len(fst) // 3
    
    # Prolazak kroz sve tRNK
    for i in range(n):
        zaglavlje = fst[3*i]
        sekvenca = fst[3*i + 1]
        struktura = fst[3*i + 2]
        
        # Provera i konstrukcija tRNK
        if proveri_unos(sekvenca, zaglavlje):
            yield tRNK(sekvenca, struktura, zaglavlje)

7. Konstrukcija modela ⮭¶

Modeli se konstruiu tako to se gramatika indukuje na osnovu skupa za obuavanje. U tom skupu se nalazi deo tRNK sa pridruenim stablima izvoenja. Iz svakog stabla mogue je izvui niz primenjenih pravila izvoenja, na osnovu kojih je nadalje mogue odrediti verovatnoe izvoenja u gramatici. Nakon obuavanja, gramatika moe posluiti za predvianje struktura, i to pomou Viterbijevog parsera, koji odreuje najverovatnije (CYK) stablo izvoenja.

In [40]:
# Biblioteka za rad sa PCFG
from nltk import induce_pcfg, Nonterminal, ViterbiParser

# Stohastika kontekstno slobodna gramatika
class Model:
    def __init__(self, trening):
        # Pravljenje i obuavanje gramatike
        self.gramatika = self.obui(trening)
        
        # Sortiranje pravila radi lepeg prikaza
        self.gramatika.productions().sort(
            key=lambda p: (p.lhs(), len(p.rhs()), p.prob()), reverse=True)
        
        # Pravljenje parsera nad gramatikom
        self.parser = ViterbiParser(self.gramatika)
    
    # Prikaz gramatike u konzoli
    def _ipython_display_(self):
        print(self.gramatika)
    
    # Pravljenje i obuavanje gramatike
    def obui(self, trening):
        # Izdvajanje svih stabala izvoenja
        trening = map(self.drvo, trening)
        
        # Izdvajanje svih pravila izvoenja
        pravila = [p for t in trening for p in t.productions()]
        
        # Indukcija gramatike nad pravilima
        return induce_pcfg(Nonterminal('S'), pravila)
    
    # Pravljenje strukture na osnovu drveta
    @staticmethod
    def od_drveta(*drvo):
        # Ukoliko je list, neuparena je baza
        if isinstance(drvo[0], str):
            if len(drvo) == 1:
                return '.'
            
            # Trolano izvoenje je uparivanje
            if len(drvo) == 3:
                return '(' + Model.od_drveta(*drvo[1]) + ')'
        
        # Sva ostala izvoenja se razlau
        return ''.join(Model.od_drveta(*d) for d in drvo)
    
    # Predvianje jedne strukture
    def predvidi(self, test):
        # Izdvajanje sekvence testne RNK
        sekvenca = test.sekvenca
        
        # Formiranje stabla izvoenja
        stablo = self.parser.parse_one(sekvenca)
        
        # Formiranje strukture prema stablu
        return Model.od_drveta(*stablo)

    # Evaluacija uspeha gramatika
    def mere(self, struktura, oznaka):
        # Udeo jednakih elemenata strukture
        jednakost = [i == j for i, j in zip(self.struktura, struktura)]
        jednakost = sum(jednakost) / len(jednakost)
        
        # Mape uparenosti na osnovu strukture
        _, stvarni = proveri_strukturu(self.struktura)
        _, predvidjeni = proveri_strukturu(struktura)
        
        # Skupovi parova prema uparenostima
        stvarni = set(stvarni.items())
        predvidjeni = set(predvidjeni.items())
        
        # Odreivanje odziva i preciznosti
        presek = len(stvarni & predvidjeni)
        odziv = presek / len(stvarni)
        preciznost = presek / len(predvidjeni)
        
        # Popunjavanje odgovarajuih atributa
        setattr(self, f'{oznaka}_jed', jednakost)
        setattr(self, f'{oznaka}_odz', odziv)
        setattr(self, f'{oznaka}_prec', preciznost)
        setattr(self, f'{oznaka}_isti', preciznost == 1)
        
        # Odreivanje niske za prikaz na slici
        niska = f'[jednakost: {jednakost:.0%}] [odziv: {odziv:.0%}] [preciznost: {preciznost:.0%}]'
        setattr(self, f'{oznaka}_niska', niska)

Ipak, znaajno je uvesti i neke numerike mere uspenosti. Jednostavna, ali moda previe optimistina mera, jeste udeo tano predvienih oznaka, odnosno udeo pogoenih karaktera (take ili zagrade). Njega je mogue izraunati na nivou pojedinanih RNK, a zatim uproseiti nad celim skupom za proveru. Druga, analogna, ali moda previe pesimistina mera, jeste udeo sasvim pogoenih struktura na celom skupu. Ove mere uzimaju u obzir celu sekvencu.

Alternativno, problem predvianja sekundarne strukture moe se shvatiti kao problem pretraivanja informacija. U tom sluaju, informacijom (dokumentom) smatra se podatak da su dve baze uparene. Drugim reima, informacija je par $(i, j)$, pri uslovu $i < j$, to oznaava da su uparene baze na pozicijama $i$ i $j$. Relevantne mere uspenosti ovde su odziv (osetljivost, senzitivnost, udeo stvarno pozitivnih, TPR) i preciznost (pozitivna prediktivna vrednost, PPV).

9. Zaključne odredbe ⮭¶

Ribonukleinska kiselina se najee predstavlja kao niska nad azbukom ribonukleotida, koja ponekad sadri i modifikovane baze. To je njena primarna struktura. Transportna RNK pritom ima karakteristinu sekundarnu strukturu u obliku deteline sa tri lista. Ona je predstavljena niskom koja se sastoji od take (neuparene baze) i zagrada (uparivanja). Razmotrena je primena probabilistikih kontekstno slobodnih gramatika (PCFG) u predvianju uvijanja tRNK.

Opisane su i implementirane tri PCFG koje se mogu upotrebiti u svrhe predvianja: Nusinov, KH-99 i svojevrsni model kovarijacije (CM). Prokomentarisane su karakteristike, prednosti i mane svake. Zatim je preuzet skup tRNK iz baze tRNAdb sa pridruenim strukturama, nakon ega su na delu podataka obuene predstavljene gramatike. Gotovi modeli iskorieni su za predvianje sekundarne strukture drugog dela podataka, to je posluilo za evaluaciju modela.

Rezultati su uporeeni kako vizuelno, tako i upotrebom numerikih mera uspenosti karakteristinih za istraivanje podataka: udeo sasvim tano predvienih (elemenata) struktura, odziv, preciznost. Najbolje se pokazao ujedno i najsloeniji CM, sa preko pola tanih struktura, a ostalim merama bliskih maksimumu. Dobro se pokazala i gramatika KH-99, dok ona tipa Nusinov ima samo teorijski znaaj. Svi zakljuci su u saglasnosti sa rezultatima iz srodne literature.
